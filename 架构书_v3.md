## 三、逐模块契约

### 01_skeleton（原 01_skeleton.py，不变）

- **输入**：`00_scdet/events.json` 或 `00_scdet/raw_cuts.json`
- **处理**：从切点列表构建镜头骨架。两个切点之间的帧区间 = 一个 Shot。每个 Shot 的 `head`/`tail` 是简单算术（头 3 帧、尾 3 帧），`representative_frame` 是区间中间帧——这些只是占位值，真正的代表帧精选由 02 完成。
- **输出**：`skeleton.json`（含 `shots[]`，每个 shot 有 `range`、`head`、`tail`、`representative_frame`）
- **契约**：
  - 只做帧号计算，不做任何视觉/语义判断
  - `range` 采用闭区间 `[start, end]`（end 帧本身属于该 shot）
  - 不得读取任何 ASR 相关文件

### 02_select_frames（原 02_select_frames.py，不变）

- **输入**：01 的 `skeleton.json`（`shots[]` + `range` + 视频路径）
- **处理**：
  1. 一次 GPU 解码全视频 224×224（NVDEC）
  2. 全局 MAFD 曲线：逐帧计算与下一帧的平均像素差异（downscale 到 160×90 后 GPU 批量计算）
  3. 逐镜头 MAFD 低谷区检测：gaussian 平滑 → 35% 分位阈值 → 连续低于阈值的区间 = 稳定段
  4. 稳定段内 GPU 评分：稳定性（MAFD 越低越高）0.5 + 锐度（Laplacian variance）0.3 + 亮度有效（避开全黑/全白）0.2
  5. 每个稳定段选评分最高的帧 = `key_frame`；所有 key_frame 中评分最高的 = `representative_frame`
- **输出**：`skeleton.json`（更新 `shots[].representative_frame` + `shots[].key_frames`）
- **契约**：
  - 不做任何 ASR / 文本处理
  - `key_frames` 按帧号升序排列
  - `representative_frame` 必须是 `key_frames` 中的一员

### 03_dino_cluster（原 04_dino_cluster.py，不变）

- **输入**：`skeleton.json` + 02 的 key_frames
- **输出**：`shot_embedding_mean.npy` (N, 1024), `shot_visual_graph.npy` (N, N)
- **契约**：
  - 不读取任何 ASR 相关文件
  - 输出矩阵对称、对角线 = 1.0

### 04_scene_merge（原 06_graph_merge.py，重写）

- **输入**：`shot_visual_graph.npy`（**仅此一个信号源**）
- **处理**：
  1. Union-Find 在 `WINDOW=30` 内只建边，不产生最终 scene
  2. 每个 component 按 `shot_id` 排序，检查**序号连续性**：逐一递增、无跳号；不连续则在跳号处拆开
  3. 对拆分后的每个连续区间跑三级验证（mean > 0.70, p10 > 0.55, max_gap < 30），不通过则在最弱点继续拆
  4. 边权重 = **纯 visual**（删除 `TEXT_W` 项，`VIS_W` 恒为 1.0）
- **输出**：`scenes[] = [{id, shot_ids, range: {start, end}}]`
- **契约**（运行时强制 assert，一行都不能省）：

```python
# 04 出口自检
scenes.sort(key=lambda s: s["range"]["start"])

# C1: 无重叠
for i in range(len(scenes) - 1):
    assert scenes[i]["range"]["end"] <= scenes[i + 1]["range"]["start"], \
        f"C1 失败: scene[{i}] end={scenes[i]['range']['end']} > scene[{i+1}] start={scenes[i+1]['range']['start']}"

# C2: 无跳号（shot 序号连续性）
all_shot_ids = []
for s in scenes:
    all_shot_ids.extend(s["shot_ids"])
assert sorted(all_shot_ids) == list(range(max(all_shot_ids) + 1)), \
    f"C2 失败: 存在跳号，shot_ids 不连续"

# C3: shot 唯一归属
assert len(set(all_shot_ids)) == len(all_shot_ids), \
    f"C3 失败: 存在重复归属的 shot"

# C4: frame_range 与 shot_ids 一致
for s in scenes:
    shot_starts = [shots[sid]["range"]["start"] for sid in s["shot_ids"]]
    shot_ends = [shots[sid]["range"]["end"] for sid in s["shot_ids"]]
    assert s["range"]["start"] == min(shot_starts), \
        f"C4 失败: scene[{s['id']}] range.start 与 shot_ids 不一致"
    assert s["range"]["end"] == max(shot_ends), \
        f"C4 失败: scene[{s['id']}] range.end 与 shot_ids 不一致"

# C5: 不得 import/读取 asr 或 text_graph
```

> **关于连续性检查的精确约定**：`frame_range` 采用**闭区间** `[start, end]`（即 end 帧本身属于该 scene）。因此相邻 scene 的无重叠断言为 `scene[i].range.end <= scene[i+1].range.start`——用 `<=` 而非 `<`，因为两个 scene 共享边界帧是合法的（该帧同时属于前一个 scene 的最后一帧和后一个 scene 的第一帧）。

### 04.5_fragment（原 06.5_fragment.py，修改合并方向）

- **处理**：`< 3s` 短 scene 向左右哪边**视觉相似度**更高就并入哪边

- **契约**：

### 06_asr_aggregate（新增模块，取代原 05_text_cluster.py 的角色）

- **输入**：04.5 最终 `scenes[]` + 05 的 `asr_timeline.json`
- **处理**：每个 VAD 段，计算与各 scene 的时间重叠时长
  - `max_overlap > 0` → 归入重叠时长最大的 scene，**整段归入，不切字**
  - `max_overlap == 0` → 标记为孤儿段，**不得强行分配给最近的 scene**（禁止写成 `min(scenes, key=distance)` 这种"最近邻硬塞"逻辑）
  - 同一 scene 收到的多个 VAD 段按时间顺序拼接
- **输出**：`scenes[].asr_text`（挂回 scene 对象）+ 孤儿段单独输出，例如 `{"start_ms":..., "end_ms":..., "text":..., "assigned_scene_id": null}`
- **契约**：

```python

# C6: 输出文本不得出现半个字（即不可来自字符级比例切分残留）
for s in scenes:
    for seg in s.get("asr_segments", []):
        # 检查是否来自比例切分：原 VAD 段文本必须完整出现
        pass  # 具体校验方式见第六节金样本

# C7: 一个 VAD 段只能唯一归属一个 scene 或标记为孤儿
# （在聚合逻辑中自然满足，这里只做计数断言）

# C8: interior_orphan == 0
# 04 修好后 scene 在覆盖区间内逐帧连续无空隙，
# 落在区间内的 VAD 段必然能归属。
# 真出现"内部孤儿"基本等于 fps 换算 bug。

```

> **orphan_ratio 作为回归指标**：边界孤儿（片头/片尾无镜头覆盖的音频）是正常的，应单独统计为 `boundary_orphan_ratio`。`interior_orphan_ratio` 必须 = 0，任何大于 0 的值直接视为 fps 铁律违规。

### 08_event_merge（原 08_chapter.py 第一级 + exp5_event_merge_v2.py 收编）

- **输入**：07 输出的 `scenes[]`（含 profile + asr_text）
- **处理**：线性 pass，相邻两两比较，THR = 0.17
- **merge_score 公式（敲死，不可改）**：

```
total = 0.35 * topic_sim
      + 0.25 * visual_sim
      + 0.20 * characters_sim
      + 0.10 * environment_sim
      + 0.10 * asr_sim
```

- **各分量定义**：

| 分量 | 计算方式 | 备注 |
|------|---------|------|
| `topic_sim` | 两个 scene 的 `profile.topic` 列表做 embedding cosine | 用 Qwen3-Embedding-0.6B |
| `visual_sim` | 两个 scene 的 shot 间 `shot_visual_graph` 均值 | 直接读 04 输出的 N×N 矩阵 |
| `characters_sim` | 两个 scene 的 `profile.characters`（字符串）的 Jaccard | characters 是字符串不是数组 |
| `environment_sim` | 两个 scene 的 `profile.environment` 的 Jaccard | |
| `asr_sim` | 两个 scene 的 `asr_combined` 的 embedding cosine | 06 输出的整段 ASR，不是碎片 |

- **不包含的项**：
  - `event` 字段：exp5 代码里算了但没进 total，是死代码，不纳入
  - `temporal` 权重：已确认删除，不恢复
  - 无 ASR 独立项之外的"ASR 关键词"判断

- **输出**：`events[]`
- **契约**：
  - 只做相邻两两比较（线性 pass）
  - **不建 N×N 矩阵，不用图/CC 结构**
  - 不得 import `hac_merge` 或 `main()`（exp5 中的坏死代码）

### 09_chapter（原 08_chapter.py 第二级，拆分为独立文件）

- **输入**：08 输出的 `events[]`（含 scene_ids、profile、asr_text、range）
- **处理**：相邻 Event 两两综合判断是否归入同一 Chapter。判断依据涵盖以下维度的**综合证据**，不是简单的关键词匹配：

  | 维度 | 信号来源 | 作用 |
  |------|---------|------|
  | 话题一致性 | VLM `profile.topic` + ASR 文本关键词统计 | 判断两个 Event 是否在说同一件事 |
  | 场景连续性 | VLM `profile.environment` / `location_types` | 判断是否在同一地点 |
  | 人物连贯性 | VLM `profile.characters` | 判断人物是否延续 |
  | 叙事逻辑 | VLM `profile.story_role` + `event` | 判断叙事功能是否连贯（开场→推进→高潮→收尾） |
  | 时间连续性 | `range` 帧号顺序 | 相邻 Event 在时间上是否自然衔接 |
  | 视觉连贯性 | 07 VLM 的 `visual_summary` | 画面内容是否过渡自然 |

  合并规则：相邻两个 Event 的综合相似度超过阈值 → 合并。具体阈值和权重由实现时根据实测确定，**不做 N×N 全局矩阵，不做图/CC 聚类**。
- **输出**：`chapters[]`（含 `scene_ids`、`event_ids`、`range`、`title`）
- **契约**：
  - 只做相邻两两比较（线性 pass）
  - **不建 N×N 矩阵，不用图/CC 结构**
  - 综合信号来源必须同时覆盖 VLM profile 和 ASR 文本，不得只用单一信号源

## 四、产物 Schema（全 Pipeline）

> 以下为运行时 pydantic / dataclass 校验的字段级契约。每个模块输出后必须 validate，字段缺失或类型不符当场 raise。

```python
# schemas.py（建议单独建文件，每个模块 import 后 validate）

from dataclasses import dataclass, field
from typing import List, Optional, Dict, Any

@dataclass
class Shot:
    id: int
    range: Dict[str, int]          # {"start": int, "end": int}
    head: List[int]
    tail: List[int]
    representative_frame: Optional[int]
    # 以下字段在特定步骤存在，不得假设始终存在
    asr_text: Optional[str] = None
    text_valid: Optional[bool] = None
    asr_contextual: Optional[str] = None
    text_cluster: Optional[Dict] = None
    visual_cluster: Optional[Dict] = None
    key_frames: Optional[List[int]] = None

@dataclass
class Scene:
    id: int
    shot_ids: List[int]
    range: Dict[str, int]          # {"start": int, "end": int}
    representative_frame: Optional[int]
    asr_text: Optional[str] = None
    asr_segments: Optional[List[Dict]] = None
    key_frames: Optional[List[int]] = None
    profile: Optional[Dict[str, str]] = None  # 8 字段，见下方

@dataclass
class ProfileSchema:
    environment: str = ""          # 场景环境
    characters: str = ""           # 人物描述（字符串，不是数组）
    actions: str = ""              # 正在发生的动作
    objects: str = ""              # 关键物体
    event: str = ""                # 事件描述
    story_role: str = ""           # 叙事功能
    topic: str = ""                # 主题（1-5字）
    visual_summary: str = ""       # 一句话总结（10字以内）

@dataclass
class Event:
    id: int
    scene_ids: List[int]
    range: Dict[str, int]
    merge_score: Optional[float] = None
    profile: Optional[Dict] = None

@dataclass
class Chapter:
    id: int
    scene_ids: List[int]
    event_ids: Optional[List[int]] = None
    range: Dict[str, int]
    title: str = ""
    location_types: List[str] = field(default_factory=list)
```

## 五、本次重构红线（禁止事项）

1. **04 / 04.5（Scene 层）不得以任何形式引用 `asr_text` 或 VAD 数据**。04.5 原本用 ASR 关键词判断短 scene 合并方向，这次必须改成纯视觉相似度。这是最容易被 agent 悄悄"优化回去"的一条，验收时重点查。
2. **05（ASR）不得输出 shot 级映射或 N×N 矩阵**，只能是时间轴数组。
3. **08 / 09（Event/Chapter 层）不得用图/CC 结构做合并**，只能线性 pass + 相邻判断。
4. **04 输出前必须跑无重叠断言**，不能只在注释或文档里写"应合并重叠 scene"。
5. **全局铁律延续**（已在 V15.1 验证过，重写文件时容易被 agent 无意间冲掉，这里重申）：
   - 任何 `s.get("fps", DEFAULT)` 都是 bug，一律 `skeleton["fps"]`
   - 每个吃 GPU 的脚本开头 `assert torch.cuda.is_available(), "CUDA 不可用，检查 torch 安装"`，不允许静默掉到 CPU

## 六、Agent 防线（三层 + 施工纪律）

### 为什么会"吞源码"——有量化，不是玄学

- **越长越吞**：GPT-4o 翻译 200 行以下代码省略率仅 1.67%，约 700 行时飙升到 47.4%，省略概率随行数上升
- **会"优化式"删无关代码**：即便 prompt 明确禁止重构，模型仍会删掉与任务无关的代码段，推测是为压缩输出体积，并因此导致测试失败
- **删了还不报错**：LLM 被训练成总会返回点什么，跑完不抛异常却给出空的、截断的或错的结果，这类**静默失败**最难排查
- **改坏了自己也看不出**：自审能否发现行为漂移本身是未决问题，**自审不能当安全网**
- **会假报完工**：agent 会在环境状态显示任务未完成时，依然高置信度宣称已经做完

### 三层防线

#### 防线 1：契约即断言（已在做，补全缺口）

把第三节的每个"契约"变成代码里的 `assert` 语句，放在模块出口。这是最便宜的一层——写 assert 不需要额外工具，但能把 90% 的"跑通了但逻辑是错的"拦在模块出口。

**断言覆盖矩阵**：

| 模块 | 必须有的 assert | 当前状态 |
|------|---------------|---------|
| 01 | 无 ASR 引用、frame_range 与 cuts 一致 | 0 条 |
| 02 | key_frames 非空且升序、representative_frame ∈ key_frames | 0 条 |
| 04 | C1-C5（5 条） | 0 条 |
| 04.5 | C1-C5（复用 04） | 0 条 |
| 06 | C6-C8（3 条） | 0 条 |
| 08 | 权重和 = 1.00、THR 固定、无图结构 | 0 条 |
| 05 | 无 shot 级文本输出 | 0 条 |

**目标**：每个模块出口 assert 数 ≥ 3。

#### 防线 2：金样本回归（characterization test）

重构前，先把当前行为按一组固定输入冻成"金样本"（baseline output）。agent 改完后自动比对，任何行为漂移立刻现形。

**具体做法**：

1. 选 2-3 个短视频（<1min，跑完全 Pipeline <5min）作为金样本
2. 跑完后把每个模块的产出保存为 `golden/04_scenes.json`、`golden/06_asr.json` 等
3. 任何改动后跑 `pytest tests/test_golden.py`，比对当前输出与金样本
4. 改动的模块允许指标漂移（如 merge 数变化），但**未改动的模块必须逐字节一致**

```python
# tests/test_golden.py（骨架）
import json, pytest

GOLDEN_DIR = "tests/golden"

def test_04_scenes_unchanged():
    with open(f"{GOLDEN_DIR}/04_scenes.json") as f:
        golden = json.load(f)
    with open("04_scene_merge/skeleton.json") as f:
        current = json.load(f)
    assert len(current["scenes"]) == len(golden["scenes"]), "scene 数量漂移"
    # 未改动的模块：逐字节一致
    # 改动的模块（如 04 本次改了合并逻辑）：只 assert 结构性不变量

def test_06_no_overlap():
    with open("06_merge/skeleton.json") as f:
        data = json.load(f)
    scenes = data["scenes"]
    for i in range(len(scenes) - 1):
        assert scenes[i]["range"]["end"] <= scenes[i + 1]["range"]["start"]
```

#### 防线 3：边界 Schema 校验

用 pydantic / dataclass 在模块边界强制校验输入输出格式。上游少字段 / 类型不符当场 raise，不让脏数据流进下一步。

```python
# schemas.py 中的 validate 函数
from pydantic import BaseModel, ValidationError

class SceneSchema(BaseModel):
    id: int
    shot_ids: list[int]
    range: dict[str, int]
    asr_text: str | None = None
    profile: dict | None = None

def validate_scenes(scenes: list[dict]):
    for s in scenes:
        SceneSchema(**s)  # 不符合直接 raise ValidationError
```

### 施工纪律

以下规则写在 `CLAUDE.md` 或项目 `.claude/settings.json` 中，作为 agent 的硬约束：

1. **一次只解锁一个模块**。不要让 agent 在一次会话里囊括多个模块的重写，范围越大它"发挥自己理解"的空间越大。

2. **动手前先 checkpoint**。每个 agent session 开始前：
   ```bash
   git checkout -b feat/04-scene-merge
   git add -A && git commit -m "checkpoint: before 04 rewrite"
   ```

3. **改完先看 diff，再跑测试**：
   ```bash
   git diff --stat           # 先看动了哪些文件
   git diff <file>           # 逐文件审
   # 不要 git add .
   # 用 git add -p 逐 hunk 审
   ```

4. **原子提交**。feature、重构、配置不要揉成一个 squash 补丁。每个改动独立提交，方便 bisect 和单独回退。

5. **明令禁止破坏性操作**。在 prompt 或 settings 中明确写：
   - 禁止删除任何未明确要求删除的文件
   - 禁止 rename 字段（`range` → `frame_range` 是高频踩坑点）
   - 禁止 `pip install torch`（版本由 setup 脚本锁定）
   - 禁止 `git add .`（必须逐 hunk 审）

6. **不信 agent 自述**。它说"改完了"不算，assert + 金样本 + schema validate 过了才是真相。
